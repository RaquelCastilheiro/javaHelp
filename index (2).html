Os quatro pilares da Programação Orientada a Objeto (POO)

Olá! Aqui você irá aprender sobre os quatro pilares da Programação Orientada a Objeto. Vamos começar?


Encapsulamento

Quando transformamos um procedimento que apresenta muitas regras de negócio ou rotinas complexas em um método, chamamos isso de encapsulamento. Esse é um processo de esconder todos os detalhes de um objeto que não contribuem para as suas características essenciais.

Se existem partes essenciais num objeto, podemos dizer que eles possuem uma parte interna, que são os dados (que não podem ser acessados por outros objetos do sistema) e outra externa, que são os métodos (que permitem a alteração ou verificação de seu estado). Um donut, por exemplo, pode representar o objeto. Não se pode chegar a seu recheio (dados) sem passar por sua massa (métodos).

Para fazer o encapsulamento, é necessário declarar os atributos de uma classe como "private". Por exemplo, se você tem uma classe "Pessoa", seu atributo "Nome" deve ser declarado como private, caso queira encapsulá-lo.

Exemplo:

class Pessoa {
private:
string nome;
//…
};

Mas o que são classes, atributos, objetos e métodos? Objetos representam "coisas" do mundo real ou do domínio de algum problema (exemplo: o carro na garagem).
As classes representam todos os tipos de objetos (exemplo: classe carro).
Os atributos são as características gerais definidas em uma classe, como cor, tamanho, nome, ordem, entre outros.
Já os métodos são sub-rotinas que são executadas por um objeto ao receber uma mensagem. Eles determinam o comportamento dos objetos de uma classe em uma programação. Podemos dizer que um carro é um objeto, onde suas características são seus atributos (dados atrelados ao objeto) e seus comportamentos são ações ou métodos.


De volta ao encapsulamento, para acessar os atributos e os métodos encapsulados é preciso usar métodos públicos chamados getters e setters, que permitem acessar e modificar os atributos encapsulados de forma controlada. Por exemplo, na classe “Pessoa” com o atributo “Nome” encapsulado, pode-se criar métodos públicos “getNome” e “setNome” para acessar e modificar esse atributo.


Exemplo:

class Pessoa {
private:
string nome;
void setNome(string nome);
public:
string getNome();
void setNome(string nome);
//…
};



Importância e benefícios do encapsulamento

O encapsulamento oferece vários benefícios, como proteção de dados, modificabilidade, reutilização de código, testabilidade e segurança. Ele permite que os detalhes de implementação de um objeto sejam alterados sem afetar o restante do sistema, tornando o código mais flexível e fácil de manter. Com isso, os objetos podem ser reutilizados em diferentes partes do sistema sem que haja preocupação com os detalhes de implementação, além de terem as informações protegidas de acessos indesejados ou de modificações acidentais.

Herança

Herança é o mecanismo que permite criar uma nova classe baseada em uma classe existente. A nova classe (classe derivada ou subclasse) herda atributos e métodos da classe existente (classe base ou superclasse), podendo adicionar novos atributos e métodos ou modificar os existentes. Aqui você pode ter se confundido com encapsulamento, mas não se preocupe. Ao longo do conteúdo você poderá esclarecer melhor, além de poder conferir as dúvidas frequentes ao final do conteúdo.

Como exemplo, podemos ter uma superclasse Animal e duas subclasses Gato e Cachorro.

As classes Gato e Cachorro são subclasses da classe Animal e por isso elas herdam os atributos e método da sua superclasse.

Exemplo em java:

public class Animal {
public void comer() {
System.out.println("Animal está comendo");
}
}
public class Cachorro extends Animal {
public void latir() {
System.out.println("Cachorro está latindo");
}
}

Importância e benefícios da herança

Com a herança, é possível reutilizar um código já existente em uma nova classe. Também, ela facilita a adição de novas funcionalidades com base nas que já estavam implementadas.

Polimorfismo

Com polimorfismo, um objeto pode ser tratado de diferentes formas dependendo do contexto. Em POO, isso geralmente se refere à capacidade de um método de se comportar de diferentes maneiras com base no objeto que o convoca. Em outras palavras, é um princípio onde as classes derivadas de uma única classe base são capazes de invocar os métodos que, embora apresentem a mesma assinatura, comportam-se de maneira diferente para cada uma das classes derivadas. Assim como uma ferramenta pode ter diferentes funções dependendo de como é usada, métodos polimórficos podem se comportar de formas diferentes dependendo do objeto.

Exemplo:

public class Animal {
public void fazerSom() {
System.out.println("Animal faz um som");
}
}
public class Cachorro extends Animal {
public void fazerSom() {
System.out.println("Cachorro late");
}
}
public class Gato extends Animal {
public void fazerSom() {
System.out.println("Gato mia");
}
}
public class TestePolimorfismo {
public static void main(String[] args) {
Animal a;
a = new Cachorro();
a.fazerSom(); // Saída: Cachorro late
a = new Gato();
a.fazerSom(); // Saída: Gato mia
}
}


Aqui, o método fazerSom é polimórfico, pois seu comportamento muda conforme o tipo do objeto (Cachorro ou Gato).

Importância e benefícios do polimorfismo

Com o Polimorfismo, os mesmos atributos e objetos podem ser utilizados em objetos distintos, porém, com implementações lógicas diferentes. Assim, ele não só promove a reutilização de código, como também contribui para a criação de sistemas mais limpos, mais compreensíveis e mais preserváveis.

Abstração

Na abstração, os detalhes complexos são ocultos e apenas as funcionalidades essenciais são exibidas. Consiste em esconder os detalhes de algo, no caso, os detalhes desnecessários. No mundo real, utilizamos abstrações o tempo todo. Tudo que não sabemos como funciona por baixo dos panos pode ser considerado uma abstração. Um bom exemplo é quando levamos o carro para manutenção e ele volta funcionando. Mesmo que não saibamos os processos disso, o que importa é que agora ele está funcionando.

É interessante destacar alguns pontos importantes dentro da abstração. É necessário que cada objeto tenha uma identidade única, para que não haja conflito dentro do sistema, e também propriedades que os definam. Além disso, os objetos podem ter métodos que representem as ações que eles podem realizar.

Exemplo:

No exemplo, depositar e sacar são ações diferentes. Ao depositar, o saldo da conta é incrementado com o valor. Em contrapartida, o saldo é decrementado ao sacar determinado valor. Em ambos os casos, eles abstraem o que as outras operações realizaram no passado. Por outro lado, a consulta de saldo não movimenta a conta. Ela apenas exibe o saldo. A consulta abstrai os depósitos e saques feitos previamente. O foco é apenas mostrar o valor disponível no momento.

Importância e benefícios da abstração

A abstração reduz a complexidade do sistema ao ocultar detalhes internos e expor apenas o que é necessário, permite que os desenvolvedores se concentrem no que o objeto faz, em vez de como ele faz, além de criar sistemas mais organizados.
